## 2장. 변수와 자료형

### 핵심 개념
- `+`는 **같은 타입끼리만** 연결 가능. 문자열 + 정수는 `TypeError` 발생 → `str()`로 형변환 필요.
- `,`(쉼표)는 타입을 자동 변환해서 출력하고, 값 사이에 **공백 1칸을 자동으로** 추가.
- 문자열 `*` 정수는 문자열을 그대로 반복 이어붙임 (사이 공백 없음).
- `not`은 True/False를 반전시키는 스위치.

| 문제 | 오답 → 교정 |
|---|---|
| `name + "는 " + age + "살이에요"` | age(int)를 그대로 `+`에 써서 에러 발생한다는 점을 놓침 → `str(age)` 필요 |
| `"파이썬" * 3` | "파이썬 파이썬 파이썬"(공백 O)로 오답 → 실제는 `"파이썬파이썬파이썬"`(공백 없음) |

## 3장. 연산자와 함수

### 핵심 개념
- **`/` vs `//` vs `%`**
  - `/` : 정밀한 소수 몫 (예: `17/5 = 3.4`)
  - `//` : 정수 몫만, 소수점 버림 (예: `17//5 = 3`)
  - `%` : 나머지만 (예: `17%5 = 2`)
  - `//`의 결과 타입은 피연산자를 따라감 (실수 하나라도 섞이면 결과도 float)
- **`range()` vs `randrange()`**
  - `range()`: 파이썬 내장, 순서대로 나열되는 구간 생성 (반복문용, 무작위 아님)
  - `randrange()`: `random` 모듈 소속, 그 구간에서 무작위로 값 하나 뽑음
  - 인자 1개면 시작값은 자동 0 (`range(3)` = `range(0,3)` = 0,1,2)
  - 둘 다 **끝 숫자는 포함하지 않음** (`randrange(1,10)` → 1~9)
- **`randrange()` vs `randint()`**
  - `randrange(a,b)` : a ~ b **직전까지** (b 미포함)
  - `randint(a,b)` : a ~ b **모두 포함**
- **`from math import *` vs `import math`**
  - `import *` : 짧지만 이름 충돌(namespace collision) 위험 — 같은 이름 함수가 여러 모듈에 있으면 나중 것이 조용히 덮어씀 (에러 없이 틀린 결과)
  - `import math` (+ `math.함수()`) : 출처가 명확, 실무 표준 스타일

### 파이썬 공통 규칙 (직접 도출한 인사이트)
> `range`, `randrange`, 문자열 슬라이싱(`[a:b]`)은 전부 **"끝 값은 울타리 바깥(미포함)"** 이라는 동일한 규칙을 공유한다. 길이 계산 시 `b - a`만으로 개수가 바로 나오게 하려는 설계.

### Q&A 하이라이트
| 문제 | 오답 → 교정 |
|---|---|
| `17 / 5` | `3.0`으로 오답 → `//`와 혼동, 실제는 `3.4` |
| `result//=6; result%=3; print(result)` | 중간값을 다 출력한다고 착각 → `print()`가 마지막 한 줄뿐이므로 최종값 `0`만 출력 |

## 4장. 문자열 다루기

### 핵심 개념
- **인덱싱/슬라이싱**: `문자열[i]`는 한 글자, `문자열[a:b]`는 a부터 b 직전까지. `[:n]`/`[n:]`/`[-n:]` 응용.
- **`.find(sub, start)`**: 못 찾으면 `-1` 반환 (에러 아님). 두 번째 인자는 **탐색 시작 인덱스**(그 위치 포함)이지, 결과값이 아님.
- **`.index(sub, start)`**: 못 찾으면 `ValueError` 발생 (프로그램 중단).
- **`.count(sub)`**: 등장 횟수. **`len()`**: 전체 문자 개수(공백·기호·숫자 포함, 의미 해석 없이 전부 카운트).
- **`.isupper()`**: 문자열을 바꾸는 함수가 아니라 "전부 대문자인가?"를 True/False로 **판정만** 하는 함수 (`.upper()`와 혼동 주의).
- **문자열은 불변(immutable)**: `.replace()` 등은 원본을 바꾸지 않고 새 문자열을 반환만 함. 재할당(`x = x.replace(...)`) 안 하면 원본 그대로 유지.
- **포매팅 3형제**: `%d/%s/%c/%f`, `.format()`(순서/인덱스/키워드 지정 가능), f-string(`f"{변수}"`, 실무 표준).
- **탈출문자**: `\n`(줄바꿈, 최다 사용) / `\t`(탭, 다음 8칸 단위로 이동) / `\r`(커서를 줄 맨 앞으로, 진행률 표시에 사용) / `\b`(커서 한 칸 왼쪽 이동 후 덮어쓰기, 실무에선 거의 안 씀) / `\\`, `\"`, `\'`(따옴표·백슬래시 자체 출력, 경로 처리에 실사용).

### 4장 최종 학습 문제
url = "http://daum.net"
my_str = url.replace("http://", "")
my_str = my_str[:my_str.index(".")]
password = my_str[:3] + str(len(my_str)) + str(my_str.count("e")) + "!"
print("{0}의 비밀번호는 {1}입니다.".format(url, password))

2. my_str.index(".")가 몇을 반환하고, my_str[:그값]으로 뭘 잘라내는지
3. password 조합에서 각 조각(my_str[:3], str(len(my_str)), str(my_str.count("e")), "!")이 각각 뭔지

## 오늘 배운 것 중 실무 연결 포인트
1. `find()` vs `index()` 선택 기준 = "없는 게 정상 상황인가, 버그 신호인가" → 예외 처리 설계 사고방식의 기초.
2. `import *`의 이름 충돌 위험 → 실무에서 와일드카드 import를 지양하는 이유 (Java의 `import package.*`도 동일한 문제).
3. 문자열 불변성(immutable) 개념 → 원본 보존이 필요할 때 별도 변수로 분리하는 습관 (최종 실습에서 `url`과 `my_str`을 분리한 이유).
4. `range`/슬라이싱의 "끝 미포함" 규칙은 파이썬 전역에서 일관되게 적용되는 설계 원칙.

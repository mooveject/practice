## 5장. 자료구조

### 핵심 개념
- **리스트**: `append(값)`(맨 뒤 추가), `insert(인덱스, 값)`(위치 지정 삽입), `pop()`(맨 뒤 값을 반환하며 삭제, 인덱스 지정 가능), `clear()`(전체 삭제).
- **제자리 수정형 vs 새 결과 반환형** (이번 복습의 핵심 개념)
  - 제자리 수정형: `.sort()`, `.append()`, `.insert()`, `shuffle()` 등 → 원본을 직접 바꾸고 **반환값은 `None`**
  - 새 결과 반환형: `sorted()`, `sample()` 등 → 원본은 그대로, **정렬/추출된 새 리스트를 반환**
  - `x = 리스트.sort()`처럼 제자리 수정형의 결과를 재할당하면 `x`는 `None`이 되는 흔한 버그 주의
- **딕셔너리**: `.get(key, default)`로 안전하게 조회(키 없으면 에러 대신 `None` 또는 지정한 default), `in` 연산자는 **기본적으로 key만 검사**(value를 검사하려면 `.values()` 명시), `.keys()`/`.values()`/`.items()`, `del 딕셔너리[key]`.
- **튜플**: 값 변경/추가/삭제 불가(읽기 전용), 여러 변수 동시 할당 및 `(a, b) = (b, a)` 스왑에 활용.
- **세트**: 중복 불허, 순서 없음. 교집합 `&`/`.intersection()`, 합집합 `|`/`.union()`, 차집합 `-`/`.difference()`.
- **`shuffle()` vs `sample()`**: `shuffle()`은 원본 리스트 순서를 직접 섞고 `None` 반환(제자리 수정형), `sample(리스트, n)`은 원본을 안 건드리고 무작위로 n개를 뽑은 **새 리스트**를 반환.

### Q&A 하이라이트
| 문제 | 오답 → 교정 |
|---|---|
| `print(subway.pop())` | 리스트 전체를 출력한다고 착각 → `pop()`은 **제거된 값 하나만** 반환 |
| `3 in cabinet`, `"피글렛" in cabinet` | `get()`과 혼동해 값 자체를 답함 → `in`은 True/False만 반환, 딕셔너리에선 **key만** 검사 |
| `cabinet.get(5)` | `none`(소문자)으로 오답 → 파이썬의 `None`은 대문자 |
| `result = shuffle(nums)` 이후 `result`, `nums` | 뒤바꿔서 오답(뒤섞인 값을 result로, 원본을 nums로 착각) → 실제는 `result=None`(shuffle은 반환값 없음), `nums`가 직접 뒤섞임 |
| `a = nums.sort()` 이후 `a`, `b=sorted(nums)`, `nums` | `a`를 정렬된 리스트로 오답 → `.sort()`는 원본을 정렬하고 `a`는 `None` |
| `x = lst.append(9)` 이후 `x`, `lst` | 역시 `x`를 리스트로 오답 → `append()`도 제자리 수정형, `x`는 `None` |

## 6장. 제어문

### 핵심 개념
- **if / elif / else**: 조건을 위에서부터 순서대로 검사, 처음 만족하는 블록만 실행.
- **for**: `for 변수 in 반복대상:` — 리스트, `range()` 등에서 값을 하나씩 꺼내 반복.
- **while**: 조건이 참인 동안 반복. 무한 루프 주의(`while True:` + 탈출 조건 필요).
- **`continue`**: 이번 반복만 건너뛰고 **다음 반복은 계속 진행**(반복문 자체는 안 끝남).
- **`break`**: **반복문 전체를 즉시 종료**. 이후 남은 반복 대상은 아예 검사되지 않음.
- **리스트 컴프리헨션(한 줄 for문)**: `[결과식 for 변수 in 반복대상]` — 일반 for문 + append 4줄을 한 줄로 압축.

### Q&A 하이라이트
| 문제 | 오답 → 교정 |
|---|---|
| `continue`/`break`가 섞인 for문 결과 예측 | `break` 이후에도 남은 학생들이 출력된다고 착각 → `break`는 반복문 자체를 완전히 끝내서 이후 대상은 아예 미실행 |
| `continue`가 걸린 숫자(3)를 출력 목록에 포함 | `continue`는 그 반복의 이후 코드(예: `print`)를 건너뛰므로 **출력 안 됨** |

## 7장. 함수

### 핵심 개념
- **매개변수(parameter)와 반환값(return)**: 함수 호출 시 값이 복사되어 매개변수로 전달되고, `return`으로 결과를 밖에 돌려줌. **재할당(`x = 함수(x)`)을 안 하면 반환값은 사라지고 바깥 변수는 그대로 유지됨** — 실무에서 정말 흔한 버그 지점.
- **변수는 "코드 실행 순서"에 따라 최신 값을 유지**: 이전 줄에서 재할당된 값이 다음 줄에 그대로 이어짐(순차적 실행).
- **매개변수 기본값**: `def profile(name, age=20):` — 기본값 있는 매개변수는 없는 매개변수보다 뒤에 위치해야 함.
- **키워드 인자 vs 위치 인자**: 위치 인자는 순서대로, 키워드 인자(`age=20`)는 순서 무관하게 지정 가능.
- **가변 인자(`*매개변수`)**: 개수가 정해지지 않은 값들을 전부 **튜플**로 묶어서 받음 (리스트 아님).
- **지역변수 vs 전역변수**: 함수 안에서 전역변수와 같은 이름으로 값을 대입하면, 그건 전역변수와 **완전히 별개인 새 지역변수**가 생성됨. 함수가 끝나면 지역변수는 사라지고 전역변수는 영향받지 않음. (`global` 키워드로 전역변수를 직접 수정할 수 있지만 실무에서는 지양, `return`으로 값을 돌려받는 방식 선호)

### Q&A 하이라이트
| 문제 | 오답 → 교정 |
|---|---|
| `deposit(balance, 500)` 호출 시 왜 balance가 1000부터 시작하는지 | 바로 위 줄에서 `balance = deposit(balance, 1000)`으로 이미 재할당되어 최신값(1000)을 물려받은 것 — 코드는 순차 실행되고 변수는 항상 최신 상태를 반영 |
| `add_ten(x)` 두 번째, 세 번째 호출 결과 | 세 번째 호출을 35로 오답 → 두 번째 호출도 재할당 안 해서 `x`는 여전히 15, 그래서 세 번째도 두 번째와 똑같이 25 |
| `*language`의 자료형 | 값은 맞혔지만 타입/형태를 놓침 → `*가변인자`는 **튜플**로 묶임 (`('파이썬','자바','C')`, `<class 'tuple'>`), 리스트 아님 |
| `total(*nums)`의 `nums` 타입 | `<class 'int'>`로 오답 → 여러 숫자를 담은 **튜플**, `sum()`은 리스트든 튜플이든 숫자 묶음이면 다 적용 가능 |
| 지역변수 vs 전역변수 (`glasses` 예제) | 세 줄 출력 전부 정답: 함수 안 지역변수는 전역변수와 별개라는 점을 정확히 적용함 |

## 오늘 배운 것 중 실무 연결 포인트
1. **제자리 수정형(반환값 `None`) vs 새 결과 반환형** 함수 구분은 실무에서 가장 흔한 버그(`x = 리스트.sort()`처럼 써서 `x`가 `None`이 되는 것) 원인 — 함수 이름이 "~해라"(동사, 원본을 직접 바꿈)인지 "~된 것을 달라"(원본 보존, 새로 반환)인지로 구분하는 습관.
2. **`in` 연산자와 딕셔너리**: key만 기본 검사한다는 점 — 실무에서 "값이 존재하는지" 확인할 때 key/value 중 뭘 검사하는지 명확히 해야 버그를 피함.
3. **함수 반환값 재할당 누락**은 실무에서 "함수는 호출했는데 왜 값이 안 바뀌지?" 하는 흔한 디버깅 포인트 — 함수가 `return`을 하면, 그 결과를 실제로 변수에 담아야(`x = f(x)`) 반영된다는 점을 항상 확인.
4. **지역변수/전역변수 스코프**: 여러 함수가 뒤섞인 코드에서 "같은 이름인데 다른 변수"인 상황이 버그의 흔한 원인 — `global` 키워드보다 `return`으로 값을 명시적으로 주고받는 설계가 실무에서 더 선호됨.

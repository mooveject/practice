## 문제 1 (2~3장: 쉼표 출력 / `+`와 `str()` / `//`, `%`)

```python
name = "다나"
age = 7
weight = 22.567

print(name, age, "살이고 몸무게는", round(weight, 1), "kg입니다.")
print(name + "는 " + str(age) + "살입니다.")
print(age // 2, age % 2)
```

**내 답변:**
```
다나 7 살이고 몸무게는 22.6 kg입니다.
다나는 7살입니다.
3, 1
```

**결과:** 전부 정답.

**개념 탐색:**
- `print(a, b, c, ...)`처럼 쉼표로 나열하면 타입에 상관없이 자동 출력되고, 값 사이에 공백이 자동으로 한 칸씩 들어간다.
- `round(22.567, 1)`은 소수점 첫째 자리까지 반올림 → `22.6`.
- `+`는 문자열끼리만 연결 가능하므로 `age`(int)는 `str(age)`로 변환해야 함.
- `//`는 몫(버림), `%`는 나머지 → `7 // 2 = 3`, `7 % 2 = 1`.

## 문제 2 (4장: 슬라이싱 / `.lower()` / `.replace()` / `.count()` / f-string)

```python
word = "Data Science"

print(word[5:])
print(word.lower())
print(word.replace("Data", "Big"))
print(word.count("a"))
print(f"저는 {word}를 공부합니다.")
```

**내 답변:**
```
Science
data science
Big Science
2
저는 Data Science를 공부합니다.
```

**결과:** 전부 정답.

**개념 탐색:**
- 인덱스 0부터 세면 공백도 한 칸으로 포함되어 `word[5]`가 `"S"` → `word[5:]`는 `"Science"`.
- `.replace()`는 원본을 바꾸지 않고 새 문자열을 반환(문자열은 불변).
- `.count("a")`는 대소문자를 구분해서 셈 → 대문자 `D`는 카운트 대상 아님, 소문자 `a`만 2번.
- f-string은 변수를 문자열 안에 그대로 삽입하는 실무 표준 포매팅 방식.

## 문제 3 (5장: 딕셔너리 값 추가/수정 + 튜플 언패킹)

```python
fruits = {"apple": 3, "banana": 5}
fruits["cherry"] = 7
fruits["apple"] += 2

print(fruits)
print(sum(fruits.values()))
print("banana" in fruits)

numbers = (10, 20, 30)
a, b, c = numbers
print(a + c)
```

**내 답변:**
```
5, 5, 7
17
True
40
```

**결과:** 계산값은 전부 정답. 다만 `print(fruits)`의 실제 출력 형식을 값만 나열한 것으로 답해서 보완이 필요했음.

**오답을 통한 개념 탐색:**
- `fruits["cherry"] = 7`은 없던 key `"cherry"`를 새로 추가하는 것, `fruits["apple"] += 2`는 기존 key의 value를 덮어쓰는 것(3+2=5).
- `print(fruits)`의 **정확한 출력은 딕셔너리 전체 형태**: `{'apple': 5, 'banana': 5, 'cherry': 7}` — 값만 나열되는 게 아니라 key까지 포함된 딕셔너리 자체가 출력됨.
- **추가 질문:** `fruits["cherry"] = 7`에서 대괄호 `[]`가 리스트 인덱스처럼 보여서 헷갈림 발생.
  - **정리한 개념:** 대괄호 `[]`의 의미는 앞에 있는 변수의 자료형에 따라 달라짐. 리스트라면 `[]` 안은 "정수 인덱스"(순서), 딕셔너리라면 `[]` 안은 "key"(이름표). `fruits`는 `{}`로 만든 딕셔너리이므로 `fruits["cherry"]`는 key 접근/추가를 의미.
- `a, b, c = numbers`는 튜플 언패킹 — `(10,20,30)`을 순서대로 각 변수에 자동 분배. `a+c = 10+30 = 40`.

## 문제 4 (6장: 조건부 리스트 컴프리헨션 + `continue` 누적 카운트)

```python
scores = [55, 82, 91, 40, 76]
passed = [s for s in scores if s >= 60]
print(passed)

count = 0
for s in scores:
    if s < 60:
        continue
    count += 1
print(count)
```

**내 답변:**
```
[82, 91, 76]
1
1
1
```

**결과:** `passed` 리스트는 정답. `count` 출력은 오답 — `print(count)`가 세 번 출력된다고 답함.

**오답을 통한 개념 탐색:**
- `[s for s in scores if s >= 60]`은 조건을 만족하는 값만 골라 새 리스트를 만드는 조건부 리스트 컴프리헨션 → `[82, 91, 76]` (정답).
- **핵심 오해:** `print(count)`가 `for` 반복문 **안**에 있다고 착각해서, `count += 1`이 실행될 때마다(3번) 매번 출력된다고 생각함.
- **실제 코드 구조:** `print(count)`는 들여쓰기상 `for`문 **밖**에 있으므로, 반복문이 전부 끝난 뒤 딱 한 번만 실행됨.
- `count`는 조건(`s >= 60`)을 만족할 때마다 `+1`씩 누적되어 55→(skip), 82→1, 91→2, 40→(skip), 76→3 순으로 쌓여 최종 `count = 3`.
- 정답: `print(count)` → `3` (한 번만 출력, "1"이 세 번 아님).
- 이전 3장 복습에서 다뤘던 `result //= 6; result %= 3; print(result)`와 동일한 패턴 — "중간 계산 과정이 아니라, print가 실제로 실행되는 위치와 횟수"를 기준으로 출력을 판단해야 함.

## 문제 5 (7장: 기본값 + 가변 인자 혼합 함수, 반환값)

```python
def make_profile(name, level=1, *skills):
    print("이름: {0}, 레벨: {1}".format(name, level))
    print("보유 스킬:", skills)
    return level * 10

exp = make_profile("루나", 3, "검술", "마법")
print(exp)
```

**내 답변:**
```
이름 : 루나, 레벨 : 3
보유 스킬 : 검술 마법
(세 번째 줄 누락)
```

**결과:** 이름/레벨 출력은 정답. `skills` 출력 형식과 `print(exp)`(반환값) 두 가지를 놓침.

**오답을 통한 개념 탐색:**
- 함수 호출 시 값 배정: `name="루나"`, `level`은 기본값 1이 있지만 위치 인자로 3을 넘겨서 **덮어씀**, 남은 `"검술", "마법"`은 `*skills`로 몰려서 **튜플**로 묶임.
- **오해 1 — `skills` 출력 형식:** `print("보유 스킬:", skills)`에서 `skills`는 튜플이므로, 출력도 튜플 형태 그대로 `('검술', '마법')`로 나옴. 공백으로만 나열된 `"검술 마법"`이 아님 — 이전 7장 복습에서 확인했던 "`*가변인자`는 튜플로 묶인다"는 개념을 출력 형식에도 그대로 적용해야 함.
- **오해 2 — 반환값 누락:** `return level * 10`은 `3 * 10 = 30`을 반환하고, `exp = make_profile(...)`로 재할당했으므로 `exp = 30`. 마지막 `print(exp)`는 `30`을 출력해야 하는데 이 줄 자체를 놓침 — 함수의 **반환값도 코드의 일부**이니 마지막 줄까지 빠짐없이 확인하는 습관 필요.
- 최종 정답:
  ```
  이름: 루나, 레벨: 3
  보유 스킬: ('검술', '마법')
  30
  ```

## 5문제를 통해 새로 드러난 공통 포인트
1. **`print()`가 실제로 어디에 위치하고 몇 번 실행되는지**를 코드 들여쓰기와 구조로 정확히 판단하는 습관 (문제 4).
2. **딕셔너리/튜플이 `print()`될 때는 자료형 고유의 표기 형태(`{}`, `()`) 그대로 출력**된다는 것 — 값만 풀어서 나열되는 게 아님 (문제 3, 5).
3. **대괄호 `[]`의 의미는 변수의 자료형(리스트 vs 딕셔너리)에 따라 달라진다**는 것 (문제 3).
4. **함수의 반환값(`return`)도 코드 실행 결과의 일부**이므로, 함수 호출 이후 이어지는 `print(반환값)` 줄까지 빠짐없이 확인해야 한다는 것 (문제 5).
